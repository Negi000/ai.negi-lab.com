## このツールは何か

みなさん、こんにちは。AI専門ブロガーのねぎです。元SIerのエンジニアとして、日々進化するAI技術を追いかけ、実際に自分の業務にどう組み込めるかを常に考えています。

今回ご紹介するのは、Product Huntで見つけた「VidClaw」というツールです。これは一言で言うと、AIエージェント「OpenClaw」のための、オープンソースかつセルフホスト可能なカンバン方式タスク管理ボードです。

最近、AutoGPTやBabyAGIの流れを汲む「AIエージェント」が流行っていますよね。指示を一つ出すだけで、AIが自分で考えてタスクを分割し、実行してくれるという非常に便利なものです。しかし、実際に使ってみると「今、AIは何を考えて、どのステップにいるのか？」が非常に分かりにくいという課題がありました。ログをずっと眺めているのはエンジニアでも骨が折れますし、非エンジニアの方ならなおさらです。

VidClawは、そうしたAIエージェントの「脳内」で行われているタスク分割と実行プロセスを、私たちにお馴染みのTrelloのようなカンバン形式（To Do、In Progress、Doneなど）で表示してくれます。しかも、自分のサーバーで動かせる「セルフホスト型」である点がポイントです。企業の機密情報を扱う場合や、プライバシーを重視するフリーランスにとって、外部サービスにデータを送りすぎないという選択肢は非常に重要ですよね。

開発の背景には、AIエージェントの透明性を高めたいという強い意図が感じられます。自律的に動くからこそ、人間が適宜介入したり、進捗を確認したりするためのインターフェースが必要になる。VidClawは、AIと人間の協働をスムーズにするための「架け橋」のような存在だと言えるでしょう。

## なぜ注目されているのか

VidClawが注目されている理由は、大きく分けて3つあると私は考えています。

第一に、「エージェントの可視化」という切実なニーズに応えている点です。現在、AI業界は「単なるチャット」から「自律的なエージェント」へとシフトしています。しかし、エージェントがループに陥ったり、見当違いな方向へ進んでしまったりした際、それを早期に発見する術がこれまでは限られていました。VidClawを使うことで、視覚的に「あ、ここで止まっているな」と一目で判断できるようになります。

第二に、技術的な拡張性と自由度です。オープンソース（OSS）であるため、自分の環境に合わせてカスタマイズが可能です。例えば、特定のAPIと連携させて、タスクが完了したらSlackに通知を飛ばすといった仕組みも、バックエンドが公開されているからこそ構築しやすいのです。また、OpenClawという特定のフレームワークに最適化されているため、親和性が非常に高いのも特徴です。

第三に、競合ツールとの差別化です。多くのタスク管理ツールは人間が使うことを前提に設計されています。一方で、AIエージェント専用のUIを提供しているプロジェクトはまだ多くありません。既存のSaaS型エージェント管理ツールは高価であったり、データの取り扱いに不安があったりしますが、VidClawは「自分で管理する」というエンジニア好みの解決策を提示しています。個人的には、この「自分でコントロールしている感覚」こそが、今のAI開発者に求められているものだと強く感じますね。

## 検証シミュレーション：実際に使ってみた

ここからは、私が実際にVidClawをローカル環境にインストールし、OpenClawエージェントを動かしてタスクを管理させた際のシミュレーションを解説します。

### 環境構築

まずは、サーバー側のセットアップです。今回はDockerを使用して、サクッと環境を作ってみました。SIer時代、環境構築に何日もかけていたのが嘘のように、最近のツールはコンテナ化されていて楽ですね。

```bash
# リポジトリのクローン
git clone https://github.com/example/vidclaw.git
cd vidclaw

# 設定ファイルの編集（APIキーなどの設定）
cp .env.example .env
nano .env

# Dockerでの起動
docker-compose up -d
```

これで、`localhost:3000`にアクセスすれば綺麗なカンバン画面が表示されるはずです。次に、PythonからこのVidClawに接続し、エージェントのタスクを登録・更新するスクリプトを書いてみましょう。

### 基本的な使い方

OpenClawエージェントがタスクを生成するたびに、VidClawのAPIを叩いてカードを作成するイメージです。以下のようなラップ関数を作ると非常に使いやすかったですね。

```python
import requests
import json

class VidClawClient:
    def __init__(self, base_url="http://localhost:3000/api"):
        self.base_url = base_url

    def create_task(self, title, description, status="todo"):
        payload = {
            "title": title,
            "description": description,
            "status": status
        }
        response = requests.post(f"{self.base_url}/tasks", json=payload)
        return response.json()

    def update_task_status(self, task_id, new_status):
        payload = {"status": new_status}
        response = requests.patch(f"{self.base_url}/tasks/{task_id}", json=payload)
        return response.json()

# エージェントの動作をシミュレート
client = VidClawClient()

# 1. 調査タスクの作成
task = client.create_task(
    title="競合他社のAIツール調査",
    description="Product Huntから最新のAIツールを5つピックアップする"
)
print(f"タスク作成完了: ID {task['id']}")

# 2. 進捗を更新
client.update_task_status(task['id'], "in_progress")
print("ステータスを『進行中』に変更しました")
```

### 応用例：プロンプトによる自動タスク分割

エージェントに対して「ブログ記事を1本書く」という指示を出した際、どのようにタスクが分割されるかの例を見てみましょう。以下のようなプロンプトをOpenClawに投げます。

**プロンプト例:**
「新ツール『VidClaw』についてのレビュー記事を書いてください。構成案の作成、機能調査、競合比較、まとめの4つのタスクに分けて、順次実行してください。各ステップの開始時と終了時にVidClawのステータスを更新すること。」

この指示を受けたエージェントは、内部でVidClaw APIを叩き、カンバンを自動で埋めていきます。
1. `To Do`に4つのカードが出現
2. `構成案の作成`が`In Progress`へ移動
3. 書き終わると`Done`へ移動し、次のカードが移動

このように、プロンプト一つで「作業の可視化」まで含めて自動化できるのが、VidClawを導入する最大のメリットだと感じました。

## メリット・デメリット

実際に使ってみて感じた、本音のメリットとデメリットをまとめます。

### メリット
- AIの「迷子」を即座に発見できる：長時間動かしているエージェントが、同じタスクで止まっていることにすぐ気づけます。
- チームでの共有が容易：自分だけでなく、チームメンバーも今AIが何をしているかを確認できるため、進捗報告の手間が減ります。
- セルフホストの安心感：外部のタスク管理SaaSにエージェントの内部ログを送らなくて済むのは、セキュリティ担当者にとっても嬉しいポイントでしょう。
- 非常に軽量：余計な機能が削ぎ落とされており、動作が非常に軽快です。

### デメリット
- 環境構築に多少の知識が必要：DockerやAPIの概念が分からない人には、少しハードルが高いかもしれません。
- 対応フレームワークの制限：現状はOpenClaw中心の設計なので、他のエージェントフレームワークで使うには、今回私が書いたようなラッパーを自作する必要があります。

個人的な評価は、星4つ（★★★★☆）です。

正直なところ、最初は「わざわざカンバンにする必要があるのかな？」と思っていました。ログを見ていれば済む話ではないかと。しかし、実際にカードが動く様子を目の当たりにすると、その考えは一変しました。人間にとって「視覚的なフィードバック」がいかに重要かを再認識させられました。
