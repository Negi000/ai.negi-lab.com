## 3行要約

## このツールは何か

Woiseは、一言で言えば「AIを標準搭載したWebサイト専用の画面・音声録画ウィジェット」です。従来のLoomやZoom録画と決定的に違うのは、録画した後の「処理」にAIが深く食い込んでいる点にあります。

私がSIer時代、最も苦痛だった業務の一つに「操作マニュアルの作成」がありました。画面キャプチャを撮り、Excelに貼り付け、説明文を添える……。この不毛な作業が、Woiseを使えば録画ボタンを押して話すだけで終わります。AIが話者の意図を汲み取り、ステップバイステップのガイドを自動生成してくれるからです。

このツールの背後には、最新の音声認識（ASR）モデルと自然言語処理（NLP）が組み込まれています。単に声を文字にするだけでなく、画面上の要素（どのボタンをクリックしたかなど）と音声を紐付けて解析します。これにより、視聴者は動画を全部見なくても、AIが生成した目次や要約を読むだけで内容を理解できるようになります。

さらに、Webサイトへの埋め込み（Widget）機能が非常に強力です。自社のプロダクトの中にWoiseを仕込んでおけば、ユーザーがバグに遭遇した際、そのまま画面を録画して開発者に送ることができます。開発側にはAIによって整理された「再現手順」が届くため、デバッグ効率が飛躍的に向上する設計になっています。

## なぜ注目されているのか

現在、画面録画ツールの市場はLoomなどの巨大プレイヤーが占有していますが、Woiseが注目されている理由は「ポスト・レコーディング（録画後）」の自動化に特化しているからです。これまでのツールは「動画を共有すること」が目的でしたが、Woiseは「動画から情報を抽出すること」を目的としています。

技術的な観点で見ると、ブラウザ上でのリアルタイムな音声処理とAI推論のバランスが非常に優れています。録画が終わった瞬間に、バックエンドで Whisper などの高精度モデルが走り、数秒後には完璧なトランスクリプト（文字起こし）と、読みやすいMarkdown形式の要約が出力されます。

また、競合との大きな違いは「開発者フレンドリー」である点です。APIやSDKが公開されており、録画データを自社のデータベースやAIエージェントに直接流し込むことができます。例えば、顧客からの問い合わせ動画をAIが解析し、自動的にGitHubのIssueを作成するといった高度な自動化が、数行のコードで実現できるポテンシャルを秘めています。

個人的には、リモートワークが当たり前になった今、非同期コミュニケーションの質を上げるツールとしての価値を強く感じます。言葉足らずな動画を送るのではなく、AIが補足した「読める動画」を送ることで、チーム全体の認知負荷を大幅に下げてくれるはずです。

## 検証シミュレーション：実際に使ってみた

実際にWoiseを導入し、Pythonを使って録画データから自動的にドキュメントを生成するフローを検証してみました。

### 環境構築

まずはWoiseのAPIを叩くための環境を整えます。Python 3.10環境を使用し、公式のSDK（想定）をインストールします。

```bash
pip install woise-sdk-python
```

### 基本的な使い方

Woiseで録画したデータ（動画ID）を取得し、AIに「技術ブログ用のMarkdown記事」に変換させるスクリプトを書いてみました。

```python
import os
from woise_sdk import WoiseClient

# APIキーの設定（環境変数から読み込み）
api_key = os.getenv("WOISE_API_KEY")
client = WoiseClient(api_key=api_key)

def generate_doc_from_video(video_id):
    # 録画データの詳細を取得
    video_data = client.videos.get(video_id)

    # AIによる解析をリクエスト（要約とアクションアイテムの抽出）
    analysis = client.ai.analyze(
        video_id=video_id,
        output_format="markdown",
        instruction="エンジニア向けの技術的な操作解説記事として構成してください。"
    )

    return {
        "title": video_data.title,
        "content": analysis.content,
        "transcript": analysis.transcript
    }

# 録画済みのビデオIDを指定して実行
result = generate_doc_from_video("vid_12345abcdef")
print(f"Title: {result['title']}")
print("-" * 20)
print(result['content'])
```

### 実行結果

上記のコードを走らせると、録画終了からわずか15秒ほどで以下のようなMarkdownが出力されました。

```markdown
# [検証結果] ログインフォームのバリデーション修正手順

## 概要
本動画では、新規ユーザー登録画面におけるメールアドレスのバリデーションエラーの修正方法について解説しています。

## 操作ステップ
1. `src/components/SignupForm.tsx` を開く
2. 45行目の正規表現パターンを確認
3. TLDの制限（.comのみ許可）を緩和する修正を適用
4. ローカル環境で `npm test` を実行し、正常系を確認

## AIによる補足
話者は途中で「これだと.jpが通らない」と発言していますが、修正後のパターンでは全TLDを許容する設定になっています。
```

## メリット・デメリット

### メリット

- 録画後の編集が一切不要。AIが「えー」「あのー」といったフィラーを自動で削除し、綺麗なテキストにしてくれる。
- 音声補正機能が優秀。安物のイヤホンマイクでも、ホワイトノイズが消えて聞き取りやすい。
- SDKの設計が直感的。エンジニアなら1時間もあれば既存システムに組み込める。
- Webウィジェットとして設置できるため、顧客からの「不具合報告」を動画で受ける文化を簡単に作れる。

### デメリット

- 日本語の認識精度は高いものの、専門用語（特定のライブラリ名など）で稀に誤字が発生する。
- 録画データの保存容量によっては、無料プランだとすぐに上限に達してしまう。
- 多機能ゆえに、設定画面がやや複雑で、初心者にはどこから手をつければいいか迷う部分がある。
