## このツールは何か

Unblocked Code Reviewは、単なるソースコードの静的解析ツールや、単純なAIチャットボットではありません。これは、開発チームがこれまでに蓄積してきた「ナレッジ」を理解した上で、プルリクエスト（PR）に対してインテリジェントなフィードバックを返す次世代のAIコードレビュー・ソリューションです。

開発の現場では、日々大量のコードが書かれ、レビューされています。しかし、既存のAIレビューツールの多くは、コードの書き方（シンタックス）や一般的なベストプラクティスに基づいた指摘に終始しがちでした。その結果、すでにチーム内で合意が取れている設計判断に対して「ここはこう書くべきです」といった、いわゆるノイズ（不要な指摘）が飛んでくることも少なくありません。

Unblockedが提供するのは、あえて「沈黙する時を知っている」AIです。このツールは、GitHubのプルリクエスト、Slackでの過去のやり取り、NotionやJiraにある仕様書、そしてソースコードベース全体をスキャンし、統合的なコンテキストを構築します。その上で、新しく提案されたコードが「なぜそのように書かれたのか」を推測し、過去の設計方針と矛盾する場合や、明らかなバグがある場合にのみ、適切なタイミングでアドバイスを投げかけます。まさに、チームに長年居座っているシニアエンジニアのような振る舞いを目指して開発されています。

## なぜ注目されているのか

現在、AIによるコーディング支援ツールは群雄割拠の状態です。GitHub Copilotを筆頭に、CursorのようなAIネイティブなエディタも人気を博しています。その中でUnblockedがこれほどまでに注目を集めている理由は、レビューの「質」と「納得感」に徹底的にこだわっているからです。

技術的な特徴として最も大きいのは、RAG（検索拡張生成）の高度な活用です。通常のAIレビューツールは、現在レビュー対象となっている差分（Diff）のみを見て判断します。一方、Unblockedは「なぜ半年前のPRでこのリファクタリングが見送られたのか」といった、Gitの履歴やドキュメントに隠れた背景を引っ張り出してきます。

また、競合ツールとの決定的な違いは、開発者のフローを邪魔しない設計です。従来のAIツールは、指摘が多すぎて結局人間がすべて無視してしまう「通知疲れ」を引き起こしがちでした。Unblockedは、介入すべきかどうかのスコアリングを行い、価値のある指摘のみを表示するように最適化されています。これにより、レビュープロセスの高速化だけでなく、レビュー自体の質を向上させることができるため、多くのテックリードやマネージャー層から熱い視線を浴びているのです。

ここからは、私が実際にUnblocked Code Reviewをプロジェクトに導入し、GitHubと連携させて動かしてみた様子をシミュレーション形式でお伝えします。

### 環境構築

Unblockedはクラウドネイティブなツールですが、ローカル環境からの操作やCI/CDへの組み込みもスムーズです。今回は、CLIツールを使って初期設定を行う手順を想定してみます。

```bash
# Unblocked CLIのインストール
npm install -g @unblocked/cli

# 認証とプロジェクトの紐付け
unblocked auth login
unblocked project init
```

設定画面では、NotionのワークスペースとSlackの特定のチャンネルへのアクセス権限を付与しました。これにより、AIが「チームの文脈」を学習できるようになります。

        }

        return f"TOKEN_{user_id}_{datetime.utcnow().timestamp()}"

        return True

このコードをGitHubへプッシュし、プルリクエストを作成します。

驚きました。単に「古いライブラリです」と言うだけでなく、どこでその議論があったのかというソースまで示してくれています。これなら、新しくチームに入ったメンバーでも「なぜこれがいけないのか」を一瞬で理解できますよね。

### 応用例

さらに高度な使い方として、未実装の機能に対する指摘も行ってみました。例えば、データベースの更新処理を含む関数において、リトライロジックが抜けている場合にどう反応するかを試しました。

```python
# db_handler.py
def update_user_status(db, user_id, status):
    # 直接更新処理のみを書く
    query = "UPDATE users SET status = %s WHERE id = %s"
    db.execute(query, (status, user_id))
    print(f"User {user_id} updated to {status}")
```

このPRに対して、Unblockedは以下のようなアドバイスをくれました。

```text
Unblocked Code Review によるコメント:

過去の障害レポート（Jira: OPS-1234）によると、データベース接続の瞬断による
更新失敗が頻発したため、更新系処理にはリトライデコレータ `@with_retry` を
付与することがチームの標準ルールとなっています。

この関数にも適用することをお勧めします。
```

このように、過去の失敗から得た教訓を現在のコードに反映させることができるのです。これは、個人の知識量に依存しがちなコードレビューにおいて、非常に強力な武器になると感じました。

## メリット・デメリット

### メリット

- 圧倒的な「文脈理解」能力
  過去の議論やドキュメントをベースに指摘をくれるため、的外れなコメントが非常に少ないです。
- 教育コストの削減
  ドキュメントへのリンクを添えて指摘してくれるため、シニアエンジニアが何度も同じ説明をする手間が省けます。
- オンボーディングの加速
  新入社員が「このチームの暗黙の了解」をコードを書きながら学ぶことができます。
- レビューの心理的ハードルの低下
  AIが先に基本的なコンテキストチェックを済ませてくれるため、人間はより本質的なロジックの議論に集中できます。

### デメリット

- 初期設定の重要性
  SlackやNotion、Jiraなどとの連携を適切に行わないと、Unblockedの真価を発揮できません。
- プライバシーとセキュリティへの懸念
  社内の機密情報や過去のやり取りをAIがスキャンするため、導入にはセキュリティ部門との綿密な調整が必要です。
- コスト面
  高機能である分、シンプルなAIツールと比べるとコストが高くなる可能性があります（組織規模によりますが）。

## どんな人におすすめか

Unblocked Code Reviewは、特に以下のような環境で働く方々に刺さるツールだと思います。

1. 成長中のスタートアップ
メンバーが急増し、ドキュメントの更新が追いつかなかったり、過去の経緯を知る人が少なくなったりしているチームには最適です。

2. 歴史の長いプロダクトを扱うエンジニア
「なぜここがこうなっているのか」という秘伝のタレ化したロジックが多いプロジェクトでは、AIが過去のGit履歴から文脈を掘り起こしてくれる機能が神がかって見えます。

3. レビューのノイズに疲弊しているテックリード
若手からのPRに対して、細かいコーディング規約や過去の決定事項を何度も指摘することに疲れているリーダーにとって、Unblockedは最高の副操縦士になってくれるはずです。

4. 非同期開発を重視するリモートチーム
Slackなどでの議論が散らばりがちなチームにおいて、それらを集約してレビューに活かせるメリットは計り知れません。
