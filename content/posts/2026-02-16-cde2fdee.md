## このツールは何か

みなさん、こんにちは。AI専門ブロガーのねぎです。元SIerエンジニアとして、これまで数多くの業務システムやWebサービスの開発に携わってきましたが、いつの時代も開発者を悩ませるのが「入力フォーム」の存在です。

今回ご紹介する「Formaly」は、その古くからある「フォーム」という概念を根本から変えてしまうかもしれない、非常に野心的なツールです。一言で言えば、「会話をしているかのような感覚で入力できるフォーム作成プラットフォーム」ですね。

Product Huntで見つけたとき、私は思わず「あ、これはSIer時代に欲しかったな」と独り言を漏らしてしまいました。従来のフォームといえば、四角い入力ボックスが並び、ユーザーがそれを一つずつ埋めていく……という、悪く言えば「作業」のようなものでしたよね。みなさんも、あまりに項目が多いフォームを見て、入力を諦めてページを閉じてしまった経験があるのではないでしょうか。

Formalyは、そのような「フォーム疲れ」を解消するために生まれました。ユーザーに対して一度にすべての質問をぶつけるのではなく、チャット形式で一つずつ質問を投げかけます。開発背景には、ユーザー体験（UX）の向上によるコンバージョン率（CVR）の改善という明確な目的があります。AIによる自然な言語処理をバックエンドに備えることで、単なる条件分岐を超えた「文脈を汲み取った対話」をフォーム上で実現しているのが最大の特徴です。

## なぜ注目されているのか

なぜ、いまFormalyがこれほどまでに注目を集めているのでしょうか。そこには、現在のWebマーケティングやカスタマーサポートが抱える構造的な課題が関係していると私は考えています。

技術的な側面から見ると、これまでの「Typeform」などのステップ型フォームとは一線を画す「動的なコンテクスト理解」が挙げられます。競合ツールとの大きな違いは、AIによる入力内容のリアルタイム解析です。例えば、ユーザーが「最近、業務の効率化に悩んでいて……」と自由記述で入力した際、その内容からキーワードを抽出し、次に投げるべき質問をAIがその場で判断して生成する。こうした柔軟性は、静的な定義に基づいた従来のツールでは不可能でした。

また、エンジニア視点で見れば、その拡張性の高さも見逃せません。ヘッドレスCMSのようにデータを扱うことができ、WebhooksやAPIを通じてSlack、Notion、Salesforceといった外部ツールとシームレスに連携できます。

正直なところ、これまで「会話型UI」を自前で実装しようとすると、フロントエンドの複雑なステート管理やバックエンドのNLP（自然言語処理）エンジニアリングが必要で、かなりの工数がかかっていました。それがFormalyのようなSaaSを使うことで、数分でデプロイできてしまう。このスピード感こそが、スタートアップや個人の開発者から熱狂的に支持されている理由なのだと思います。

## 検証シミュレーション：実際に使ってみた

今回は、私が運営している架空のAIコンサルティングサービスの「お問い合わせフォーム」をFormalyで作ってみる、という設定で検証を行いました。

特に、Pythonを使ってFormalyのSDK経由でフォームの挙動を制御したり、データを取得したりするプロセスをシミュレーションしてみます。

### 環境構築

まずはSDKのインストールから始めます。架空のライブラリですが、現代的なツールならこのような構成になるはずです。

```bash
pip install formaly-sdk python-dotenv
```

### 基本的な使い方

FormalyのAPIを使用して、ユーザーとの対話をシミュレーションし、回答内容を解析するコードを書いてみました。

```python
import os
from formaly import FormalyClient
from dotenv import load_dotenv

# 環境変数の読み込み
load_dotenv()
API_KEY = os.getenv("FORMALY_API_KEY")

# クライアントの初期化
client = FormalyClient(api_key=API_KEY)

def create_smart_form():
    # 新しい会話型フォームの作成
    form = client.forms.create(
        name="AI導入相談フォーム",
        theme="modern-dark",
        ai_logic=True  # AIによる動的な質問生成を有効化
    )

    # 基本的な質問の追加
    form.add_step("こんにちは！あなたのビジネスについて少し教えていただけますか？")

    # ユーザーの回答を待機するシミュレーション
    # 実際にはWeb上のUIでユーザーが入力します
    print(f"Form Created: {form.url}")
    return form

def process_submission(submission_id):
    # 送信された内容を取得
    submission = client.submissions.get(submission_id)

    # AIによる感情分析やキーワード抽出の結果を確認
    analysis = submission.ai_analysis

    print(f"ユーザーの熱量: {analysis.sentiment_score}")
    print(f"抽出された課題: {analysis.key_issues}")

    # 課題に応じて次のアクションを分岐
    if "コスト削減" in analysis.key_issues:
        send_to_sales_team(submission)
    else:
        send_to_consulting_team(submission)

# 実行（シミュレーション）
if __name__ == "__main__":
    my_form = create_smart_form()
    # 仮の送信IDでデータ処理をテスト
    process_submission("sub_12345abcde")
```

### 応用例：プロンプトによる会話のパーソナライズ

Formalyの面白いところは、フォーム内のAIに対して「システムプロンプト」を設定できる点です。これにより、フォームの性格を自由に変えることができます。

例えば、以下のようなプロンプトを設定してみたとしましょう。

「あなたは親しみやすい技術コンサルタントとして振る舞ってください。ユーザーが技術的な用語を使った場合は、それを深掘りして具体的な技術スタックを聞き出してください。ただし、あまりに専門的になりすぎないよう、柔らかい口調を保ってください。」

この設定をした後のユーザーとのやり取りは、以下のような変化を見せました。

ユーザー： 「最近はLangChainを使ってRAGの構築を試しています。」
Formaly： 「LangChainをお使いなのですね！素晴らしいです。RAGの精度向上、特にベクターデータベースの選定やプロンプトエンジニアリングの部分でお困りのことはありませんか？」

このように、入力内容に基づいて「聞き返す」ことができるのは、従来のフォームでは絶対に不可能な体験でした。

- ユーザーの心理的ハードルを下げる圧倒的なUI

- AIによる高度なセグメンテーション
回答内容をリアルタイムでAIが分析し、それに応じた質問を生成するため、ユーザー一人ひとりに最適化された体験を提供できます。

- 開発スピードの向上
ノーコードに近い感覚で構築できるため、エンジニアがいちいちReactなどで複雑なフォームコンポーネントを組む必要がなくなります。SIer時代の私が見たら、泣いて喜んでいたでしょう。

### デメリット

- 日本語の微細なニュアンスへの対応
AIベースである以上、日本語の非常に複雑な敬語表現や、業界特有の隠語に対して、時として不自然な返しをしてしまう可能性があります。ここはプロンプトの調整が必要な部分ですね。

- 従来の「一覧性」の喪失
すべての項目を一気に確認したい、というタイプのユーザーにとっては、一問一答形式はかえってストレスになる場合があります。用途に応じた使い分けが不可欠です。

- 運用コスト
高機能なAI機能を利用する場合、無料枠を超えた際のサブスクリプション料金が、シンプルなGoogleフォームなどと比較すると高めに設定される傾向があります。
